Entity-Relationship model, table schemas, column constraints, relationships, and indexes for MemoLink's PostgreSQL database with pgvector.

Entity-Relationship Overview

Five core tables: users, notes, embeddings, conversations, and messages. All user-scoped tables cascade-delete when the user is removed.

Table

users

Stores registered user accounts. The root entity — all other tables are scoped to a user via foreign keys.

👤 users PostgreSQL table
ColumnTypeConstraintsNotes
id INTEGER PKINDEX Auto-incrementing primary key
email VARCHAR UNIQUENOT NULL Unique email address used for login
password VARCHAR NOT NULL Hashed password (bcrypt or passlib)
Table

notes

Stores user-created notes. Each note is owned by one user and has at most one associated embedding (1-to-1).

📝 notes PostgreSQL table
ColumnTypeConstraintsNotes
id INTEGER PKINDEX Auto-incrementing primary key
user_id INTEGER FK → users.idNOT NULL CASCADE DELETE — removing user removes all notes
title VARCHAR(255) nullable Optional display title for the note
content TEXT NOT NULL Full body text of the note — this is embedded
source VARCHAR(255) nullable Optional label — e.g. "Lecture 3", "uploaded PDF filename"
created_at TIMESTAMPTZ server default: now() Timestamp of note creation (UTC)
Table

embeddings

Stores the 1 536-dimensional vector embedding for each note. Uses the pgvector extension. One embedding per note (unique FK). Cosine similarity search runs against this table.

🧠 embeddings pgvector table — requires CREATE EXTENSION vector
ColumnTypeConstraintsNotes
id INTEGER PKINDEX Auto-incrementing primary key
note_id INTEGER FK → notes.idUNIQUE CASCADE DELETE — one embedding per note maximum
vector VECTOR(1536) OpenAI text-embedding-ada-002 output — 1 536 float dimensions
Table

conversations

Represents a named chat session owned by a user. Contains one or more messages.

💬 conversations PostgreSQL table
ColumnTypeConstraintsNotes
id INTEGER PK Auto-incrementing primary key
user_id INTEGER FK → users.idNOT NULL Owner of this conversation
title VARCHAR(255) nullable User-defined or auto-generated label
created_at TIMESTAMPTZ server default: now() Timestamp when session was created
Table

messages

Stores individual chat messages (both user and AI) within a conversation. Supports paginated retrieval via before_id cursor.

📨 messages PostgreSQL table
ColumnTypeConstraintsNotes
id INTEGER PK Auto-incrementing primary key
conversation_id INTEGER FK → conversations.id CASCADE DELETE — messages deleted with conversation
role VARCHAR(20) Either "user" or "assistant"
content TEXT Raw message text (Markdown supported in UI)
created_at TIMESTAMPTZ server default: now() Used for ordering and paginated cursor (before_id)
Relationships

Table Relationships

  • users 1 → many notes A user owns many notes. notes.user_id → users.id (CASCADE DELETE)
  • notes 1 → 0..1 embeddings Each note has at most one embedding. embeddings.note_id UNIQUE FK (CASCADE DELETE)
  • users 1 → many conversations A user owns many conversations. conversations.user_id → users.id
  • conversations 1 → many messages A conversation contains many messages. messages.conversation_id → conversations.id (CASCADE DELETE)
