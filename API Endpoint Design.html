Auth

Authentication Endpoints

POST /api/v1/auth/register Register a new user account ▼

Creates a new user. Returns 400 if the email is already registered. Password is hashed with bcrypt (passlib) before storage.

Request Body — RegisterDTO
{
  "email":    "user@example.com",
  "password": "securepassword"
}
Response 200 — UserResponse
{
  "id":    1,
  "email": "user@example.com"
}
Error Responses
400 Email already registered
422 Validation error (missing fields)
POST /api/v1/auth/login Log in and return user record ▼

Verifies email + hashed password. Returns the user record whose id the client stores in localStorage key memolink_user for scoped requests. Returns 401 on invalid credentials.

Request Body — LoginDTO
{
  "email":    "user@example.com",
  "password": "securepassword"
}
Response 200 — UserResponse
{
  "id":    1,
  "email": "user@example.com"
}
POST /api/v1/auth/change-password Change password with current-password verification ▼

Verifies the supplied current_password against the stored bcrypt hash before updating. The new password must be at least 8 characters. Returns 401 if current_password is wrong, 400 if the new password fails length validation.

Request Body — ChangePasswordDTO
{
  "user_id":         1,
  "current_password": "oldpass123",
  "new_password":     "newpass456"
}
Response 200
{ "ok": true }
Error Responses
401 Current password incorrect
400 New password too short (<8 chars)

Notes Endpoints

Creates the note record and triggers embedding generation via the OpenAI embeddings API. HTML is stripped before embedding. The resulting vector is stored in the embeddings table.

  "user_id": 1,
  "title":   "Lecture 5 Summary",
  "content": "<p>RAG stands for...</p>",

}

  }

Chat / RAG

Chat Endpoints

POST /api/v1/chat Submit prompt — RAG answer + citations (JSON) ▼

Core RAG endpoint. Flow: 1 — Embed the prompt via OpenAI text-embedding-3-small. 2 — Retrieve top-K notes by cosine similarity. 3 — Build a context-augmented system prompt. 4 — Call GPT for the answer. 5 — Return answer + source citations. Persists both the user message and the AI reply to the conversation. For real-time streaming use /chat/stream instead.

Request Body — ChatRequestDTO
{
  "user_id":         1,
  "conversation_id": 7,
  "prompt":          "What is retrieval-augmented generation?",
  "top_k":           5
}
Response 200 — ChatResponseDTO
{
  "answer": "RAG combines retrieval...",
  "sources": [
    {
      "note_id": 42,
      "title":   "Lecture 5",
      "snippet": "RAG stands for..."
    }
  ],
  "attachments": [],
  "message_id": 103
}
POST /api/v1/chat/stream Streaming AI response via SSE (text/event-stream) ▼

Same RAG pipeline as /chat but returns Content-Type: text/event-stream. GPT is called with stream=True. Each token is forwarded immediately. nginx must have proxy_buffering off and X-Accel-Buffering: no — both are set in nginx.conf. The frontend displays a blinking cursor while streaming; the cursor disappears when done is received.

Request Body — ChatRequestDTO
{
  "user_id":         1,
  "conversation_id": 7,
  "prompt":          "Explain embeddings.",
  "top_k":           5
}
SSE Event Stream
Content-Type: text/event-stream

data: {"t":"Embed"}
data: {"t":"dings"}
data: {"t":" are"}
... one event per token ...
data: {"done":true,"id":104}
POST /api/v1/chat/upload Upload files + prompt in one multipart request ▼

Accepts multipart/form-data. Extracts text from uploaded files (PDF, DOCX, TXT), embeds them as temporary notes, then runs the full RAG pipeline with the combined context. Attachments are returned in the response.

Form Fields
conversation_id integer  (required)
prompt           string   (optional)
files            file[]   (required, multipart)
Accepted: .txt, .pdf, .docx
Response 200 — ChatResponseDTO
{
  "answer": "Based on your document...",
  "sources": [...],
  "attachments": [
    {
      "filename":     "report.pdf",
      "content_type": "application/pdf",
      "size":         204800
    }
  ],
  "message_id": 104
}
Suggest

AI Suggestion Endpoint

POST /api/v1/suggest Generate AI reminder suggestions from notes ▼

Strips HTML from the user's notes (up to 15), concatenates them, and sends the result to GPT with a reminder-extraction prompt. GPT returns a JSON array of suggestions. Each suggestion is saved to the reminders table with type="ai". The endpoint only uses the user's own notes — no external context is injected.

Request Body — SuggestDTO
{
  "user_id": 1,
  "context": "study, work",
  "source":  "notes"
}
Response 200 — SuggestResponse
{
  "suggestions": [
    {
      "text":        "Submit assignment 3",
      "description": "Due based on Lecture 5 notes.",
      "due_date":    "2026-05-25",
      "due_time":    "23:59"
    }
  ]
}

Transcribe

Transcription Endpoint

POST /api/v1/transcribe Transcribe audio/video via OpenAI Whisper ▼

Accepts a multipart file upload (MP4, M4A, WebM, MP3, WAV, MOV — maximum 25 MB). Sends the file to OpenAI Whisper and returns the plain-text transcript. Used internally by the note editor's Record button.

Form Fields
file     audio/video file (required)
language string (optional, BCP-47)
Max size: 25 MB
Response 200
{
  "text": "The meeting started at 9 AM..."
}

Video

Video Import Endpoints

POST /api/v1/video/import Import YouTube captions by URL ▼

Accepts a YouTube video URL. Uses YouTubeTranscriptApi() (instantiated object, not static call) to fetch the available captions. Returns the title and HTML-formatted transcript. No ffmpeg or yt-dlp is required — only caption data is fetched. Returns 422 Unprocessable Entity if the video has no captions available.

Request Body — VideoImportDTO
{
  "url": "https://www.youtube.com/watch?v=abc123"
}
Response 200 — VideoImportResponse
{
  "title":   "Intro to Machine Learning",
  "content": "<p>Welcome to this tutorial...</p>",
  "method":  "captions"
}
Error Responses
422 No captions available for this video
400 Invalid YouTube URL
POST /api/v1/video/upload Upload audio/video file — Whisper transcription ▼

Accepts a multipart file upload (MP4, M4A, WebM, MP3, WAV, MOV — maximum 25 MB). Intended for Zoom, Teams, and Google Meet recordings. Sends the file to OpenAI Whisper for transcription and wraps the result in HTML for direct insertion into the Tiptap note editor.

Form Fields
file audio/video file (required)
Accepted: .mp4 .m4a .webm .mp3 .wav .mov
Max size: 25 MB
Response 200 — VideoUploadResponse
{
  "title":   "recording.mp4",
  "content": "<p>The meeting started...</p>",
  "method":  "whisper"
}
Error Responses
413 File exceeds 25 MB limit
415 Unsupported file type

Workspaces

Knowledge Workspace Endpoints

All workspace operations require a user_id in the request body. The API enforces ownership — a user can only read or modify their own workspaces. Soft-deleted workspaces are excluded from all responses unless explicitly queried.

POST /api/v1/workspaces Create a new Knowledge Workspace ▼

Creates a workspace for the given user. Returns 400 if a workspace with the same name already exists for that user. The type field must be one of: Academic, Professional, Personal, Project, Other. If this is the user's first workspace, is_default is set to true automatically.

Request Body — WorkspaceCreateDTO
{
  "user_id":     1,
  "name":        "Capstone Research",
  "type":        "Academic",
  "description": "My MemoLink capstone project workspace."
}
Response 200 — WorkspaceResponseDTO
{
  "id":              1,
  "user_id":         1,
  "name":            "Capstone Research",
  "type":            "Academic",
  "description":    "My MemoLink capstone project workspace.",
  "is_default":     true,
  "last_accessed_at": null,
  "created_at":     "2026-05-21T...",
  "alert_count":    0
}
Error Responses
400 Workspace name already exists for this user
422 Validation error (missing or invalid type)
POST /api/v1/workspaces/list List all workspaces for a user with alert counts ▼

Returns all non-deleted workspaces owned by the given user_id, ordered by last_accessed_at descending (most recently used first). Each workspace includes an alert_count — the number of non-done reminders in that workspace — which drives the alert badge in the sidebar dropdown.

Request Body — WorkspaceListDTO
{ "user_id": 1 }
Response 200 — WorkspaceResponseDTO[]
[
  {
    "id":              1,
    "name":            "Capstone Research",
    "type":            "Academic",
    "description":    "...",
    "is_default":     true,
    "last_accessed_at": "2026-05-21T...",
    "alert_count":    3
  },
  {
    "id":              2,
    "name":            "Claims System",
    "type":            "Professional",
    "description":    null,
    "is_default":     false,
    "last_accessed_at": "2026-05-20T...",
    "alert_count":    0
  }
]
POST /api/v1/workspaces/get Get a single workspace by ID ▼

Returns the full workspace record for the given workspace_id. Returns 403 if the workspace does not belong to the requesting user. Returns 404 if not found or soft-deleted.

Request Body — WorkspaceGetDTO
{
  "user_id":      1,
  "workspace_id": 1
}
Response 200 — WorkspaceResponseDTO
Full workspace object (see /workspaces/list for shape)
Error Responses
403 Workspace does not belong to user
404 Workspace not found or deleted
POST /api/v1/workspaces/update Update workspace name, type, or description ▼

Partial update — any combination of name, type, and description may be provided. Rejects a name change if the new name is already taken by another workspace belonging to the same user. Updates updated_at automatically.

Request Body — WorkspaceUpdateDTO
{
  "user_id":      1,
  "workspace_id": 1,
  "name":        "Capstone 2026",
  "type":        "Academic",
  "description": "Updated description."
}
Response 200 — WorkspaceResponseDTO
Full updated workspace object
Error Responses
400 New name already exists for this user
403 Workspace does not belong to user
404 Workspace not found
POST /api/v1/workspaces/delete Soft-delete a workspace (blocked if last workspace) ▼

Sets deleted_at on the workspace. Returns 409 if this is the user's only remaining non-deleted workspace — users must always have at least one. The frontend must switch the active workspace to another before calling this endpoint if the deleted workspace was the active one.

Request Body — WorkspaceDeleteDTO
{
  "user_id":      1,
  "workspace_id": 2
}
Response 200
{ "ok": true }
Error Responses
409 Cannot delete the only remaining workspace
403 Workspace does not belong to user
404 Workspace not found
POST /api/v1/workspaces/set-active Set the active workspace and update last_accessed_at ▼

Records the user's active workspace selection. Updates last_accessed_at on the workspace row. The frontend stores the returned workspace_id alongside the user object in localStorage key memolink_user so it persists across page refreshes. On next login the client calls /workspaces/active to restore it.

Request Body — SetActiveDTO
{
  "user_id":      1,
  "workspace_id": 1
}
Response 200 — WorkspaceResponseDTO
Full workspace object with updated last_accessed_at
POST /api/v1/workspaces/active Return the currently active workspace for a user ▼

Returns the workspace with the most recent last_accessed_at for the given user. Used on login to restore the user's last workspace selection. If no last_accessed_at is set on any workspace, falls back to the workspace where is_default = true.

Request Body — ActiveWorkspaceDTO
{ "user_id": 1 }
Response 200 — WorkspaceResponseDTO
{
  "id":              1,
  "name":            "Capstone Research",
  "type":            "Academic",
  "is_default":     true,
  "last_accessed_at": "2026-05-21T...",
  "alert_count":    3
}
Error Responses
404 No workspace found for user (redirect to onboarding)

Workspace-aware existing endpoints: All note, conversation, reminder, chat, and suggest endpoints now accept an optional workspace_id field in their request bodies. When supplied, results are filtered to that workspace. When omitted, the backend resolves the active workspace via /workspaces/active logic. RAG retrieval (pgvector cosine search) is always filtered to the active workspace unless "cross_workspace": true is passed in the chat request body.
