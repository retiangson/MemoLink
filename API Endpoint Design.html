Auth

Authentication Endpoints

POST /api/v1/auth/register Register a new user account ▼

Creates a new user. Returns 400 if the email is already registered. Password is hashed before storage.

Request Body — RegisterDTO
{
  "email":    "user@example.com",
  "password": "securepassword"
}
Response 200 — UserResponse
{
  "id":    1,
  "email": "user@example.com"
}
Error Responses
400 Email already registered
422 Validation error (missing fields)
POST /api/v1/auth/login Log in and return user record ▼

Verifies email + hashed password. Returns the user record whose id the client stores for scoped requests. Returns 401 on invalid credentials.

Request Body — LoginDTO
{
  "email":    "user@example.com",
  "password": "securepassword"
}
Response 200 — UserResponse
{
  "id":    1,
  "email": "user@example.com"
}
Notes

Notes Endpoints

POST /api/v1/notes Create a new note + auto-embed ▼

Creates the note record and triggers embedding generation via the OpenAI embeddings API. The resulting vector is stored in the embeddings table.

Request Body — NoteCreateDTO
{
  "user_id": 1,
  "title":   "Lecture 5 Summary",
  "content": "RAG stands for...",
  "source":  "lecture_5.pdf"
}
Response 200 — NoteResponseDTO
{
  "id":      42,
  "user_id": 1,
  "title":   "Lecture 5 Summary",
  "content": "RAG stands for...",
  "source":  "lecture_5.pdf"
}
POST /api/v1/notes/list List all notes for a user ▼

Returns all notes owned by the given user_id, ordered by creation date descending.

Request Body — NoteListDTO
{
  "user_id": 1
}
Response 200 — NoteResponseDTO[]
[
  {
    "id":      42,
    "user_id": 1,
    "title":   "Lecture 5",
    "content": "...",
    "source":  null
  }
]
POST /api/v1/notes/get Retrieve a single note by ID ▼

Returns the full note record for the given note_id, or null if not found.

Request Body — NoteGetDTO
{ "note_id": 42 }
Response 200 — NoteResponseDTO | null
{
  "id": 42,
  "user_id": 1,
  "title": "Lecture 5",
  "content": "...",
  "source": null
}
POST /api/v1/notes/update Update note title and/or content ▼

Partial update — only provided fields are changed. If content changes, the embedding is regenerated.

Request Body — NoteUpdateDTO
{
  "note_id": 42,
  "title":   "Updated Title",
  "content": "Updated body..."
}
Response 200 — NoteResponseDTO | null
Updated note or null if not found
POST /api/v1/notes/delete Delete a note (cascades to embedding) ▼
Request Body — NoteDeleteDTO
{ "note_id": 42 }
Response 200
{ "ok": true }
POST /api/v1/notes/search Vector similarity search over notes ▼

Accepts a pre-computed embedding vector and returns the top-K most similar notes using pgvector cosine similarity. Typically called internally by ChatService.

Request Body — NoteSearchDTO
{
  "vector": [0.0231, -0.1142, ... 1536 dims],
  "top_k": 5
}
Response 200 — NoteResponseDTO[]
[
  {
    "id":      42,
    "title":   "Lecture 5",
    "content": "...",
    "source":  "lecture_5.pdf"
  }
]
Chat / RAG

Chat Endpoints

POST /api/v1/chat Submit prompt — RAG answer + citations ▼

Core RAG endpoint. Flow: 1 — Embed the prompt via OpenAI ada-002. 2 — Retrieve top-K notes by cosine similarity. 3 — Build a context-augmented system prompt. 4 — Call GPT for the answer. 5 — Return answer + source citations. Persists both the user message and the AI reply to the conversation.

Request Body — ChatRequestDTO
{
  "user_id":         1,
  "conversation_id": 7,
  "prompt":          "What is retrieval-augmented generation?",
  "top_k":           5
}
Response 200 — ChatResponseDTO
{
  "answer": "RAG combines retrieval...",
  "sources": [
    {
      "note_id": 42,
      "title":   "Lecture 5",
      "snippet": "RAG stands for..."
    }
  ],
  "attachments": [],
  "message_id": 103
}
POST /api/v1/chat/upload Upload files + prompt in one multipart request ▼

Accepts multipart/form-data. Extracts text from uploaded files (PDF, DOCX, TXT), embeds them as temporary notes, then runs the full RAG pipeline with the combined context. Attachments are returned in the response.

Form Fields
conversation_id integer  (required)
prompt           string   (optional)
files            file[]   (required, multipart)
Accepted: .txt, .pdf, .docx
Response 200 — ChatResponseDTO
{
  "answer": "Based on your document...",
  "sources": [...],
  "attachments": [
    {
      "filename":     "report.pdf",
      "content_type": "application/pdf",
      "size":         204800
    }
  ],
  "message_id": 104
}

POST /api/v1/conversation/messages Get paginated messages in a conversation ▼

Cursor-based pagination using before_id. Omit before_id to start from the most recent message. Set limit to control page size (default 10).

Request Body + Query Params
Body:
{ "conversation_id": 7 }

Query params:
?limit=10&before_id=103
Response 200 — Message[]
[
  {
    "id":         102,
    "role":       "user",
    "content":    "What is RAG?",
    "created_at": "2026-05-19T..."
  },
  {
    "id":         103,
    "role":       "assistant",
    "content":    "RAG combines...",
    "created_at": "2026-05-19T..."
  }
]

POST /api/v1/conversation/add-to-note Save a message's content as a new note ▼

Creates a new note from the provided content — typically used to save a useful AI reply back into the user's knowledge base. Triggers embedding generation for the new note.

Request Body — AddToNoteDTO
{
  "user_id": 1,
  "content": "RAG combines retrieval...",
  "title":   "AI answer: What is RAG?"
}
Response 200 — NoteResponseDTO
{
  "id":      55,
  "user_id": 1,
  "title":   "AI answer: What is RAG?",
  "content": "RAG combines...",
  "source":  null
}
