You are working on the MemoLink project.

Task:
Implement a slash-command system in the chat input. The commands will allow users to improve notes, enhance notes, summarize notes, rewrite notes naturally, update notes, append content, undo the last note modification, create reminders, generate quizzes, run multi-model discussions, read notes aloud, submit feedback, and report bugs.

Important:
Do not break existing MemoLink features such as normal chat, notes, workspaces, RAG, embeddings, reminders, feedback, conversations, authentication, file upload, transcription, streaming responses, or admin features.

Before coding:
1. Inspect the current project structure.
2. Identify the existing notes model/table, notes API/service/repository, chat input flow, reminders flow, feedback flow, and workspace handling.
3. Reuse existing patterns and architecture.
4. Do not create duplicate systems if the project already has services or endpoints for notes, reminders, feedback, or chat.

Main requirement:
When a user types a message that starts with `/`, treat it as a command.
When a user types a normal message that does not start with `/`, keep the existing chat behavior unchanged.

Command parsing rules:
- Commands are case-insensitive.
- Commands start with `/`.
- Support quoted note names for note titles with spaces.
- Support `All` where allowed.
- Support the separator `:` for commands that require extra message/instruction.
- Trim spaces safely.
- Return clear validation messages if the command format is wrong.
- All commands must be scoped to the authenticated user and active workspace.
- Never allow a user to access or modify another user’s data.

Examples:
- `/Improve All`
- `/Improve "Assessment 2 Notes"`
- `/Enhance "Capstone Notes"`
- `/Summarize All`
- `/Natural "Research Proposal Draft"`

Database update for one-level undo:
Do not create a note_versions or note_revisions table.

Undo only needs to support one previous version per note.

Add nullable undo snapshot fields directly to the existing notes table:

- undo_title TEXT or VARCHAR
- undo_content TEXT
- undo_command VARCHAR(50)
- undo_instruction TEXT
- undo_created_at TIMESTAMPTZ
- undo_available BOOLEAN DEFAULT FALSE

Purpose:
Before any command modifies an existing note, save the current note title and content into the undo fields. This allows the user to restore the note to its immediately previous state.

This is one-level undo only.
If the same note is modified again, overwrite the undo fields with the latest previous version.

Commands that must save an undo snapshot before updating a note:
- /Improve
- /Enhance
- /Natural
- /Humanize
- /Update
- /Add

Commands that do not need an undo snapshot:
- /Summarize, because it creates a new note and does not overwrite the original
- /Reminder, because it creates a reminder
- /Quiz, because it generates quiz output only
- /Discussion, because it generates discussion output only
- /Read, because it only opens or reads a note
- /Feedback, because it creates feedback
- /ReportBug, because it creates a bug report

Undo behavior:
Only support:

`/Undo (Note Name)`

Do not support `/Undo All`.

When `/Undo "Note Name"` is triggered:
1. Find the note for the current authenticated user and active workspace.
2. Check if `undo_available = TRUE`.
3. If no undo snapshot exists, return:
   `No undo version is available for this note.`
4. If an undo snapshot exists:
   - Replace the current note title with `undo_title`.
   - Replace the current note content with `undo_content`.
   - Clear `undo_title`.
   - Clear `undo_content`.
   - Clear `undo_command`.
   - Clear `undo_instruction`.
   - Clear `undo_created_at`.
   - Set `undo_available = FALSE`.
   - Regenerate the note embedding after restoring.
   - Return a confirmation message in chat.

For bulk commands such as `/Improve All` or `/Enhance All`:
- Process notes one by one.
- Each modified note gets its own undo snapshot.
- The user can undo only one note at a time using `/Undo "Note Name"`.
- Do not implement `/Undo All`.

Command behavior details:

Purpose:
Improve grammar, clarity, structure, and Markdown formatting without adding new information.

Behavior:

- Format the output as clean Markdown.
- Save undo snapshot before updating each note.
- Update the existing note.
- Regenerate embeddings after update.

Expected response:
Return a short confirmation showing which note or notes were improved.

2. `/Enhance (All | Note Name)`

Purpose:

- Regenerate embeddings after update.

Expected response:
Return a short confirmation showing which note or notes were enhanced.

- Generate embeddings for the new summary note.

New note title format:

Expected response:
Return a confirmation with the new note title.
