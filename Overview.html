Smart Source Workspace

This supporting knowledge-capture enhancement preserves original uploads and books in OneDrive, caches source files locally per device, and connects them to autosaved editable notes, extracted RAG text, recordings, drawings, handwriting, highlights, equations, and timeline history. Users can mix text and ink across the full note surface with ballpoint, pencil, marker, highlighter, brush, calligraphy, and dashed tools; use partial or whole-stroke erasing; lock scrolling; and rely on stylus-first palm rejection. Optimistic rendering and queued background persistence keep strokes visible while database IDs and cross-device annotation state synchronize.

Source tabs are enabled only for genuinely linked notes, note recording reuses Record Lecture, and legacy notes retain their existing workflow. Original files remain unchanged unless a user explicitly exports an annotated copy; source and audio binaries are not stored in the database. Browse Books supports 10–100 items per page and Calendar supports a 1–12 month layout.

The core research evaluation remains focused on contextual retrieval, source-grounded AI responses, task support, multilingual usefulness, usability, and in-app evaluation analytics.

Academic Research Topic

Research Statement

"Design and Evaluation of a Context-Aware AI Companion for Knowledge Capture, Contextual Retrieval, and Task Support in Study and Work Settings."

MemoLink is both a functional software product and an academic research artefact. The system is evaluated against measurable criteria - retrieval relevance, answer faithfulness, usability (SUS), and task completion time - to validate whether AI-augmented note retrieval meaningfully reduces cognitive load.

Product

What is MemoLink?

MemoLink lets users capture notes and documents, then interact with their own knowledge base through a conversational AI interface powered by Retrieval-Augmented Generation (RAG). Every answer is grounded in the user's own stored notes, making responses traceable and faithful rather than generically hallucinated.

📝

Knowledge Capture

Create, edit, upload, and organise notes and documents. Rich Tiptap editor with formatting toolbar, Record, Export, and Video Import buttons.

🔍

Contextual Retrieval

Semantic vector search over stored notes using OpenAI text-embedding-3-small and pgvector cosine similarity matching.

💬

Conversational AI

Multi-turn chat sessions where every response cites the source notes that informed the answer. Tokens stream to the client in real time via SSE with a blinking cursor.

🗂️

Conversation History

Persistent, paginated conversation threads - rename, soft-delete, or save AI replies back as notes.

📎

File-in-Chat Upload

Attach files directly in the chat; MemoLink extracts, embeds, and queries them in real time.

🔐

User Accounts

Registration and login with scoped data. User menu (avatar initials) with Settings, Help, and Sign Out. Change password with current-password verification.

📡

Streaming AI

Chat responses stream token-by-token via Server-Sent Events (POST /api/chat/stream). nginx disables buffering so chunks reach the browser instantly.

🎤

Voice Input

Mic button in ChatInput triggers the useRecording hook - speech is transcribed and inserted as the prompt without typing.

⏰

Reminders & AI Suggestions

Right-sidebar Reminders panel with AI-generated and manual reminders. AI suggestions are derived from the user's notes only - GPT returns title, description, due date, and due time.

🎬

Video Import

YouTube URL tab fetches captions via youtube-transcript-api. Upload File tab accepts MP4/M4A/WebM/MP3/WAV/MOV up to 200 MB - files <5 MB use OpenAI Whisper, files ≥5 MB are routed to Deepgram Nova-2 (with Whisper as fallback up to 25 MB).

🗂️

Knowledge Workspaces

Organise notes, conversations, reminders, and AI context into named, typed workspaces - Academic, Professional, Personal, Project, or Other. RAG retrieval is scoped to the active workspace. An alert badge flags workspaces with pending reminders. First-time users are onboarded with a guided workspace creation flow.

🎨

AI Image Generation

Type /image <prompt> or use natural language triggers ("draw me a …", "generate an image of …"). MemoLink calls DALL-E 3 and streams the image as a base64 data URL inside the chat - no expiry, no CDN dependency. Attribution shows "replied by DALL-E 3".

🌐

Multi-Language Translation

Translation uses a Gemini 2.5 Flash Lite quality loop: initial translation → back-translate → GPT scores similarity (0–100) → up to 3 refinement rounds if score < 85. Results are cached in the translation_cache table; a green "Cached" badge shows when a cached result is returned. A "Re-translate" button forces a fresh LLM call. The chat bubble shows the model name and accuracy score, colour-coded green/amber/red.

🤖

Model Selection

The Settings modal lets users choose their chat model across three providers - OpenAI (GPT-4o, GPT-4o Mini, GPT-4 Turbo, GPT-3.5 Turbo), Google Gemini (2.5 Flash, 2.5 Flash Lite, 2.5 Pro), and DeepSeek (DeepSeek V3, R1, Coder). Every AI reply shows a "replied by [Model]" attribution badge. If a model fails, the system automatically tries the next available model in the fallback chain: primary → GPT-4o-mini → Gemini 2.5 Flash → DeepSeek Chat. Stored model preferences using old Gemini 2.0 IDs are automatically migrated to their 2.5 equivalents.

🔎

Web Search

A globe icon in ChatInput toggles Brave Search context per message. Top web results are injected as a system context block before the LLM call, giving answers real-time grounding beyond the user's notes. Requires BRAVE_SEARCH_API_KEY.

📄

Rich File Import

Bulk upload preserves formatting: DOCX headings (H1–H4), bold/italic/underline, bullet and numbered lists, tables (with header rows), and embedded images. PDF infers headings by font-size ratio. PPTX maps slide titles to H2 and content to lists. Images are embedded as base64 <img> tags. Files up to 200 MB supported via S3 pre-signed upload when deployed on Lambda.

🔑

Password Reset

The LoginPage includes a "Forgot password?" flow. The backend sends a signed JWT reset link (1-hour expiry) via SMTP, with a console fallback when SMTP is not configured. The reset token is detected from the ?reset_token= URL parameter.

🧪

Automatic Smart Actions

Regular chat is now the only entry point. When a request clearly needs tool use - such as saving a note, creating a reminder, searching the web, or drafting from Gmail context - MemoLink routes the turn through an internal action agent automatically and streams typed tool events before the final answer.

🛡️

Admin Panel

Full-screen overlay accessible from the user menu (admin only). Four tabs: Feedback (view and manage bug reports/suggestions), Feature Flags (toggle features, set default model/language, configure per-feature minimum access levels via Regular/Plus/Pro tabs), Users (promote/demote admin status, change access level), and System Logs (view INFO/WARNING/ERROR events with level badges, source filter, expandable JSON details, pagination, and clear-all). The first registered user is auto-promoted to admin on startup.

🏳️

Feature Flags

System-wide flags stored in the feature_flags table include web search, model selection, image generation, translation, file upload, research mode, TTS, slash commands, custom API keys, video import, email integration, AI memory graph (memograph_enabled), proactive insights (proactive_insights_enabled), answer confidence (confidence_enabled), AutoPilot routing (autopilot_enabled), meeting/lecture timeline (timeline_enabled), AI study mode (study_mode_enabled), smart actions / workflow agent (workflow_enabled), default model, and default language. Each feature also has a *_min_level flag (regular / plus / pro) that controls the minimum access tier required. Admins configure both toggles and minimum levels via the Admin Panel; the frontend fetches them on login and hides disabled controls accordingly.

🏅

Access Levels

Three user tiers - Regular, Plus, and Pro - gate access to premium features. Admins assign tiers per user via the Users tab. The tier is embedded in the JWT (access_level claim) and validated stateless on each API call. The GET /api/features endpoint automatically masks disabled features based on the caller's level. Research Mode, for example, can be set to require Plus or Pro.

📚

Research Mode

A dedicated research pipeline activated by the "Research" toggle in ChatInput. Calls POST /api/research/stream; the backend validates the user's access level against the research_mode_min_level flag before running a multi-source web search + RAG synthesis pipeline that returns a cited, structured answer via SSE streaming.

🐛

Bug Reports & Suggestions

Users submit bug reports or feature suggestions via Help → Report Bug / Suggestion. Reports are stored in the feedback table with a status lifecycle (open → read → resolved). An optional SMTP notification is sent to the admin on new submissions. Admins filter and manage feedback in the Admin Panel.

📋

System Logs

All major operations - auth, transcription, translation, uploads, chat fallbacks - write structured events to the system_logs table (INFO / WARNING / ERROR). The SystemLogService provides .info(), .warning(), and .error() methods. Admins view, filter, paginate, and clear logs from the fourth tab of the Admin Panel.

🎙️

Smart Transcription Routing

Upload limit raised to 200 MB. Files <5 MB are transcribed by OpenAI Whisper. Files ≥5 MB are routed to Deepgram Nova-2 via DEEPGRAM_API_KEY (free $200 credit, no credit card); if Deepgram fails, Whisper is used as a fallback for files up to 25 MB. Routing is transparent to users; each event is logged to system_logs.

⚡

Translation Cache

Successful translations are stored in the translation_cache table keyed by SHA-256 hash of {text}::{language}. Repeated requests are served instantly with cached: true. A force field bypasses the cache for fresh LLM translation. A hit_count counter tracks reuse. The UI shows a green "Cached" badge and a "Re-translate" pill.

🔗

Chat Fallback Chain

When the selected model fails for any reason (rate limit, API error, bad key), the system automatically tries the next model: primary → GPT-4o-mini → Gemini 2.5 Flash → DeepSeek Chat. Each failure is logged as a WARNING. Only after all configured models are exhausted does the user see an error message.

✨

Note Improvement via Chat

Natural-language triggers ("improve my note X", "clean up note X", "polish note X") detected by _IMPROVE_NOTE_RE regex. MemoLink closes the note's editor tab, shows a dual-ring spinner, fuzzy-matches the note title, calls AI to improve the HTML content, auto-saves to DB, and streams back a clickable "Open Note: X →" button.

🔑

User API Keys & Custom Providers

Users can store their own OpenAI, Gemini, or DeepSeek API keys in Settings → API Keys. Keys are encrypted at rest using Fernet symmetric encryption. Additionally, users can add any OpenAI-compatible provider (Groq, Mistral, Together AI, Ollama, Perplexity, etc.) by supplying a name, base URL, model ID, and API key. Custom providers appear in the AI Model picker alongside built-in models. The selected provider's key takes priority over server keys for all requests.

🖥️

Slash Command System

14 slash commands available directly in the chat input. Type / to open the command picker with keyboard navigation (↑↓ arrows, Tab/Enter to select, Esc to cancel). Commands: /Improve, /Enhance, /Summarize, /Natural, /Humanize, /Update, /Add, /Undo, /Reminder, /Quiz, /Discussion, /Read, /Feedback, /ReportBug. Commands that take note names show a searchable note picker after Tab. Commands that take free text (/Feedback, /ReportBug, /Reminder) show a format hint popup with examples instead. Commands that modify notes save a one-level undo snapshot automatically. All commands use the currently selected AI model with user custom keys taking priority over server keys.

📝

Interactive Quiz

The /Quiz command generates an interactive quiz from any note or all notes. Questions use radio buttons (single answer) or checkboxes (multiple answers). After submission the quiz shows score, correct answers, and per-question explanations. A Save Results to Notes button creates a formatted Markdown note with the full results - questions, answers, explanations, and topics to review.

🔈

Text-to-Speech (/Read)

The /Read command opens a note in the editor, displays it in chat, and reads it aloud using the browser's built-in Web Speech API - no extra API key required. A TTS player bar appears above the chat input with: Play/Pause, Stop, ◀◀ / ▶▶ sentence navigation, and speed controls 0.75× 1× 1.25× 1.5× 2×. Markdown symbols are stripped before speaking for natural delivery. Changing speed restarts the current sentence at the new rate.

💬

Multi-Model Discussion (/Discussion)

The /Discussion command invites every configured AI model - GPT, Gemini, DeepSeek, and any custom providers the user has added in Settings → API Keys - to each share a 3-5 sentence perspective grounded in the note content. A synthesis is then generated combining all perspectives. The command uses the selected model first, then falls back through user custom keys → server providers → server default.

📧

Gmail Email Integration

Connect Gmail via OAuth 2.0 in Settings → Email. MemoLink fetches unread and important emails, batch-scores importance with GPT (1-5), and saves those scoring ≥ 3. Auto-process appends new emails to a global Email Digest note and creates reminders for emails with detected deadlines - both are global (visible across all workspaces). A Sync from Email button in the Reminders panel triggers a fresh sync on demand. Email-sourced reminders link back to the original email. Reply to any email in-app: choose a tone (Formal / Friendly / Brief), generate 3 RAG-grounded AI draft suggestions, edit, and send - reply threads correctly in Gmail via In-Reply-To + threadId. Email Draft from Chat: say "send [recipient] the [note name]" (optionally "make it nice" / "summarize it") - MemoLink uses AI intent extraction to identify the recipient, matching note, and style hint; builds the draft from the note content (AI rewrites only when a style instruction is given); and presents a Draft Email review card with editable To/Subject, a formatted body preview, a full TipTap Edit mode, and a Send button. Sent state is tracked in localStorage to prevent double-sending. Gmail receives a multipart/alternative MIME message (HTML + plain-text fallback) so emails render correctly in all mail clients. Internally, Gmail token refresh, draft send, attachment download, and email-to-note / email-to-reminder actions are now routed through a dedicated connector and service layer instead of controller-owned HTTP logic.

🧠

AI Memory Graph (MemoGraph)

A full-screen force-directed graph that visualises how the AI perceives and connects knowledge across all notes. MemoGraph uses GPT to batch-extract entities - people, topics, projects, deadlines, decisions, action items, questions, and themes - then draws edges between entities and the notes that mention them. Notes sharing entities are automatically linked with related_to edges. The canvas supports pan, zoom, and a Fit button. Each node type has a distinct colour. Entity nodes are stored in graph_nodes and relationships in graph_edges. The graph data also powers Graph-Enhanced RAG: the retrieval pipeline traverses graph edges to find notes connected to the top-K vector hits, expanding context beyond cosine similarity alone.

💡

Proactive Reminder Intelligence

An AI-powered scan that surfaces missed deadlines, incomplete action items, and urgency signals buried in the user's notes - without the user having to ask. Detection runs in two passes: a rule-based pass (filename regex flags recently uploaded unreviewed files) and a GPT batch pass (8 notes per call, each classified into missing_reminder, incomplete_actions, or urgency_signal). Insights appear in a collapsible AI Insights section in the Reminders panel, colour-coded by severity (red / amber / indigo). Each insight links directly to the source note. Dismissed insights are soft-deleted and not re-shown. Results are persisted in the proactive_insights table and refresh on each scan.

🛡️

Source-Grounded Answer Confidence

Every AI response carries a HIGH / MEDIUM / LOW / UNSUPPORTED confidence badge inline with the model attribution. The system uses a two-layer approach: the LLM self-assesses by appending a <confidence> tag to its own reply (stripped before display), and a server-side pre-computed fallback guarantees a badge even if the model omits the tag. The fallback is deterministic - UNSUPPORTED when no notes exist, LOW/MEDIUM/HIGH based on how many top-K vector results were retrieved. Confidence and reasoning are persisted to the messages table (confidence, confidence_reason columns) so historical responses retain their badges. A hover tooltip shows the reasoning. The feature is toggled via the confidence_enabled feature flag.

⚡

AutoPilot Model Routing

An intelligent, zero-latency routing layer that automatically selects the best AI model for each prompt - with no extra API calls. Rules are evaluated in priority order: Translation (language regex) → Code / Debug (any programming language name, or ≥2 code keywords) → Deep Research (analysis/research keywords) → Long Context (prompt >250 words) → Simple Query (≤12 words, stays on default). Translation and Long Context route to Gemini 2.5 Flash (large context window); Code routes to DeepSeek Coder; Deep Research to GPT-4o. Rules skip gracefully if a model's API key is not configured. A violet ⚡ AutoPilot · {Reason} chip appears inline with the model attribution, hidden for Simple Query. Controlled via the autopilot_enabled feature flag.

⏱️

Meeting / Lecture Timeline

A Timeline tab appears in the note editor for any note containing transcribed audio or video content. One GPT-4o-mini call analyses the transcript and returns: Chapters - major topic sections with timestamps (00:03:12 Assessment requirement explained); Action Items - tasks and follow-ups with the responsible person; Important Moments - key statements tagged as Decision ⚖️, Warning ⚠️, Key Point 💡, Deadline ⏰, or Question ❓. Timestamps are estimated from word position at 130 wpm (±30–90 s accuracy for lectures). Every entry has a Jump → button that uses TipTap's doc.descendants() to locate the exact phrase in the transcript and scrolls the editor to it. Results are cached in the note_timelines table; Regenerate overwrites the cache. Controlled via the timeline_enabled feature flag.

🎓

AI Study Mode

A full-screen study workspace opened from the user menu, with six tools built on demand from the active workspace's notes: Flashcards (flippable Q/A cards from a note or all notes), Quiz (interactive single/multi-choice with scoring, explanations, and save-to-note - the same engine as /Quiz), Exam Review (key concepts, definitions, important facts, likely questions, and focus topics across selected notes), Study Plan (a day-by-day plan from a goal and time frame, each day listing focus topics, tasks, and notes to revise), Weak Topics (detects recurring topics from your questions and explains each simply with a study tip), and Summary (short / medium / detailed levels with optional bullet points). All endpoints live under business/services/study_service and are gated by the study_mode_enabled feature flag.

✨

Smart Actions (Workflow Agent)

After a normal chat reply, MemoLink silently analyses the exchange (POST /api/workflow/suggest) and, when a useful follow-up is possible, renders conversational action buttons beneath the message - no mode or toggle required. Approving an action records the user's click as a chat message, executes it, and streams the result back as a saved assistant reply, so the full exchange survives a reload. Eight actions are supported: create reminder, create note, summarise workspace, search web, organise notes, suggest title, extract tasks, and prepare report outline. Non-OpenAI models fall back to GPT-4o-mini for the tool-calling step. Per-user suggestions toggle in Settings → Workflow; admin kill-switch via the workflow_enabled feature flag. After an action runs, the Notes or Reminders panel refreshes automatically.

🔄

Resend / Retry Messages

Every user message shows a Resend button (circular arrow) so the prompt can be resubmitted without retyping. Every assistant message shows a Retry button that re-runs the same prompt and streams a fresh response, replacing the previous reply in the conversation. Both buttons use the current model and workspace context.

📊

In-App Mixed-Method Evaluation

MemoLink embeds a full research evaluation framework directly in the app. Users start an optional evaluation session (EvaluationSessionModal) which activates a floating task checklist (EvaluationPanel) with 10 guided tasks. After each AI response, an EvaluationRatingBar lets users rate retrieved context relevance, answer usefulness, answer faithfulness or supported-by-notes status, citation usefulness, and answer trust. All LLM calls automatically record telemetry to evaluation_ai_metrics: model, provider, tokens, cost, response times, retrieval scores, and confidence level - with no storage of raw prompts or note content. A 30-minute active-time budget per participant is enforced via a heartbeat mechanism; admins can adjust budgets per user.

📋

Post-Session Survey

A 37-question in-app survey (SurveyModal) covers seven sections: Participant Background, Knowledge Capture, Contextual Retrieval & AI Answers, Task Support & Reminders, Usability & UX, Trust, Privacy & Ethics, and Overall Evaluation. Questions use Likert scales (1-5), single/multi-choice, and free-text formats. The survey is designed so core SUS and usability items remain required, feature-specific sections can be shown only when relevant, and the expected completion time stays around 5-8 minutes. Responses are stored in survey_responses and survey_answers tables and are editable (one submission per account). Admins can add, edit, or reset questions via the Admin Panel Survey tab.

📤

Research Data Export

The Admin Panel Evaluation tab (AdminEvaluationPanel) shows aggregate metrics: average context relevance, answer usefulness, faithfulness or supported-by-notes status, citation usefulness, trust, response time by feature, fallback rate, confidence alignment (comparing model confidence to user trust ratings), and feature usage totals. Admins can export a CSV ZIP (six tables: sessions, tasks, events, ai_metrics, ratings, feature_usage) or a JSON report with markdown summary. The Survey tab (AdminSurveyPanel) shows per-question distributions and downloads a wide-format CSV. All exports exclude raw prompts, note content, and credentials.

🔊

Text-to-Speech Reading

Any note can be read aloud via the /Read command or the play button in the note editor, using the browser's free Web Speech API (no API cost). Playback starts from the cursor position and skips the note title, so you can begin reading from any paragraph. The player bar offers play/pause, stop, ◀◀ / ▶▶ sentence navigation, and speed control (0.75×–2×), with the voice selectable in Settings. The sentence currently being spoken is highlighted live in the editor using non-destructive ProseMirror decorations - visual overlays that never modify the document. Gated by the tts_enabled feature flag.

📖

Books Library

An admin-managed library of books synced from a OneDrive folder via Microsoft OAuth. Admins connect OneDrive in the Admin Panel's Books tab, pick a folder, and sync; users then browse, borrow into My Books, and read in-app. Supported formats: PDF, EPUB, PPTX, audio, TXT, SRT/VTT captions, CBZ/CBR comics, and MOBI. The in-app reader supports pagination, reading-progress tracking that resumes where you left off, bookmarks, light/dark/sepia colour modes, and Read Aloud text-to-speech.

🖍️

Highlights & Save as Note Source

Selecting text in a book (PDF, EPUB, PPTX, TXT, captions, or MOBI) and choosing a highlight colour appends the passage to an auto-generated note titled "{Book Title} - Highlights" - double-clicking a highlight there jumps the reader back to the exact spot, and the Notes list refreshes instantly. Save as Note Source extracts a book's full text (PDF, EPUB, MOBI, PPTX, TXT, SRT/VTT) into chunked, embedded notes so its content becomes citable in AI Chat answers, processed as a durable async pending → processing → ready/failed job.

Research Evaluation

In-App Mixed-Method Evaluation

MemoLink collects quantitative and qualitative research data directly through the application, without requiring external survey tools or manual log extraction. Data collection is privacy-preserving: no raw prompts, note content, uploaded files, or credentials are ever stored in evaluation tables.

Quantitative - Request-Level Metrics

Every LLM call records to evaluation_ai_metrics:

  • Model selected vs. actual model used (fallback detection)
  • First-token latency, total response time, stream duration
  • Embedding time, retrieval time, LLM time (breakdown)
  • Retrieved note count, top-K requested, min/max/avg retrieval score
  • Citation count, source note IDs (no note content)
  • Confidence level (HIGH / MEDIUM / LOW / UNSUPPORTED)
  • Token usage (input, output, total) and estimated cost USD
  • Web search and graph RAG flags

Quantitative - User Ratings

After each AI response, the EvaluationRatingBar collects separate judgments for source-context quality and answer quality:

  • Retrieved context relevance - 1 to 5 scale
  • Answer usefulness - 1 to 5 scale
  • Answer faithfulness / supported by notes - yes / partially / no / not sure
  • Citation usefulness - 1 to 5 scale
  • Answer trust - 1 to 5 scale
  • Task difficulty - 1 to 5 scale
  • System speed - 1 to 5 scale
  • Optional free-text comment (max 1 000 chars)

Ratings are linked to the message ID and restored after page reload via GET /api/evaluation/my-ratings.

Qualitative - Post-Session Survey

37 questions across seven sections, fully editable by admins:

  • A. Participant Background
  • B. Knowledge Capture
  • C. Contextual Retrieval & AI Answers
  • D. Task Support & Reminders
  • E. Usability & User Experience
  • F. Trust, Privacy & Ethics
  • G. Overall Evaluation

Likert (1-5), single-choice, multi-choice, and free-text formats. One response per account; editable before final submission. Core SUS and usability questions are always retained, while feature-specific questions can remain optional to reduce burden.

Privacy and Budget Controls

  • No raw prompts, answers, note content, or files stored in evaluation tables
  • Error messages capped at 500 chars; comments capped at 1 000 chars
  • 30-minute active-time budget per participant (admin-adjustable)
  • Heartbeat tracks active seconds; pauses when tab is hidden
  • Budget exhaustion halts all analytics recording
  • Participant codes (P### format) provide pseudonymity
  • Feature flags: evaluation_analytics_enabled, evaluation_survey_enabled, evaluation_admin_export_enabled

Baseline Comparison

The evaluation includes at least one manual baseline task alongside the MemoLink task. Participants first complete a retrieval or follow-up workflow using ordinary notes, transcripts, or manual task tracking, then repeat the same task with MemoLink so completion time, success rate, and perceived effort can be compared.

Action-Item Gold Standard

Action-item extraction is judged against a small gold-standard set prepared from 3-5 sample notes or transcripts. Precision, recall, and F1 are reported using Precision = TP / (TP + FP), Recall = TP / (TP + FN), and F1 = 2 x precision x recall / (precision + recall).

Multilingual Caution

Multilingual output is explicitly assistive rather than certified. It should be reviewed by humans before use, and Māori or other culturally sensitive material should only be included when approved. The evaluation focuses on usefulness and clarity, not official linguistic correctness.

Core Evaluated Scope

The core evaluated scope is knowledge capture, contextual retrieval, task support, multilingual usefulness, and evaluation analytics. Gmail, graph memory, proactive insights, smart actions, text-to-speech, and custom API keys remain supporting or optional features rather than the centre of the evaluated capstone scope.

Recent Updates

June 2026 Platform Refresh

The last two development sessions focused on making MemoLink behave less like a collection of isolated prompts and more like a coherent AI companion system. The main changes were made in the chat orchestration, academic writing path, streaming contract, action UX, and connector handling.

Smart Response Engine

Chat now follows a cleaner internal flow: request analysis, mode routing, context-engine preparation, one unified primary prompt, answer generation, and optional quality review. This replaced stacked prompt fragments that were causing shallow or inconsistent replies.

Long Academic Draft Pipeline

Large paper requests are detected explicitly and drafted as a dedicated long-form path rather than a normal chat answer. The system now plans sections, streams progress, and applies an academic quality pass before the final draft is returned.

Academic Source Fusion

Research retrieval now combines Semantic Scholar, CORE, arXiv, and OpenAlex fallback. Results are deduplicated, CORE full text is reused as richer citation context, and cited papers can be saved automatically into notes for future grounded retrieval.

Typed Streaming Events

The backend and frontend now share a typed stream-event contract for message deltas, replacements, completion metadata, quiz events, note events, tool status, and workflow actions. This removed fragile raw-payload guessing in the client.

Conversation-Aware Web Search

The “Search online?” suggestion no longer retries with a vague follow-up like “latest news”. MemoLink now carries forward the actual topic from the recent conversation and uses that as the real search query.

Email and Action UX

Email draft generation now preserves conversational context across follow-up turns, and action buttons show local loading where it matters: send button, translate, save-to-note, and quiz-result note saves. This keeps status feedback tied to the action that caused it.

Implementation note: the Smart Response Engine architecture is now documented separately so the product overview does not have to carry every orchestration detail. See Smart Response Engine.html for the end-to-end workflow diagram and mode-routing explanation.

Deployment

Docker Compose Deployment

MemoLink ships with a complete Docker Compose configuration. The entire stack - FastAPI backend, React/nginx frontend, and all configuration - is brought up with a single command after copying the environment file.

Prerequisites: Docker Desktop installed; .env file populated from .env.example. 

cp .env.example .env        # fill in OPENAI_API_KEY, DATABASE_URL, DEEPGRAM_API_KEY (optional, for ≥5 MB files)
                            # also S3_UPLOAD_BUCKET, AWS_REGION (optional, for Lambda S3 pre-signed uploads)
docker compose up --build   # builds and starts backend + frontend

The backend container runs on port 8000 behind the nginx reverse proxy. nginx forwards all /api/ requests to FastAPI and sets X-Accel-Buffering: no and proxy_buffering off so SSE tokens flow through without delay. The frontend is served as a static bundle on port 80. No ffmpeg or external media tools are required in production - YouTube captions are fetched directly via HTTP and Whisper runs inside the backend container.

Engineering

Design Principles

Clean Architecture

Controllers depend only on service interfaces. No database calls in controllers. No business logic in repositories.

Domain-Driven Design

Repositories encapsulate all persistence. The domain layer is framework-agnostic and testable in isolation.

Dependency Injection

ServiceInstaller binds interfaces to concrete implementations. Controllers resolve services through the container - not manually.

Testability

All services depend on interfaces (Protocols). Repositories and the embedding service can be mocked independently.
