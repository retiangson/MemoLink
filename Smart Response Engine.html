Smart Source Integration

How Source Notes Enter AI Workflows

RAG consumes extracted source text, editor content, confirmed transcripts, saved highlights, annotation comments, and equation results—not raw OneDrive files, local recordings, or drawing binaries. Book Save as Note Source jobs extract supported PDF, EPUB, MOBI, PPTX, TXT, SRT, and VTT content asynchronously and generate searchable note chunks with source/page context.

Solve Equation and Complete Equation flush the autosaved note first. A handwritten equation may be supplied as a temporary request-scoped image only to a vision-capable model; text-only models receive text context instead. The engine validates structured output, escapes it, formats the steps for readability, and appends the durable result to the note.

Privacy boundary: source and audio binaries are never embedded or retained by the note database. OneDrive holds originals, each device owns its local cache, and MemoLink synchronizes metadata, extracted text, note content, editable annotation JSON, and timeline events.

Purpose

Why This Layer Exists

MemoLink serves very different user intents: casual chat, academic writing, email drafting, note-grounded retrieval, workflow actions, and live web search. A single static prompt is not a robust control surface for that range. The Smart Response Engine exists to make internal decisions explicit: what mode the request belongs to, what context should be loaded, which retrieval path to use, what output budget is appropriate, and what metadata should be saved when the answer is finished.

The current design goal is not “more prompt text.” It is better orchestration: route correctly, gather the right evidence, generate with one coherent primary prompt, and persist outcomes so the next turn is stronger.

What Changed

Key Upgrades from the Last Two Sessions

Unified Prompt Entry

After routing, MemoLink now builds one primary system prompt for the selected mode instead of layering separate mode, date, and optimized-task prompts as competing instructions.

Routing Prompt hygiene

Context Engine Layer

Mode-specific context preparation now lives in a dedicated context-engine layer, so academic paper retrieval and prompt assembly are not buried as inline special cases.

Context engine Separation of concerns

Long Academic Draft Path

Full-paper requests are detected and routed into a dedicated drafting workflow with planning, broader retrieval, section-based generation, and a final academic quality pass.

Academic writing Progress streaming

Parallel Academic Search

Academic retrieval now queries Semantic Scholar or OpenAlex fallback, CORE, and arXiv in parallel, then merges results in deterministic priority order. This cuts latency without making the output order unstable.

Semantic Scholar CORE arXiv

Cited Paper Auto-Save

When a draft actually cites retrieved papers, MemoLink can save those papers as notes with authors, year, DOI, abstract, and CORE full-text excerpt so future RAG retrieval is grounded in both user notes and captured literature.

Note creation Embeddings

Conversation-Aware Search and Email

Search-online retries now preserve the actual topic from recent turns, and email draft generation can recover context from follow-up turns such as sending only an email address after an earlier draft request.

Conversation context Action continuity

Connector Cleanup

Gmail transport concerns were split out into a dedicated connector so token refresh, message fetch, send, and attachment download now live behind one adapter boundary. The email service keeps orchestration responsibility for reminders, note creation, digest updates, and chat-driven draft flows.

Connector boundary Reminder persistence
Academic Flow

Academic Writing Path in Detail

The academic path is the most demanding branch because it must balance user notes, uploaded requirements, external literature, structure constraints, and large output budgets. It is now handled as a first-class workflow instead of a dressed-up normal chat call.

Stage What Happens Why It Matters
Analysis Detect full paper / rubric / minimum word-count style requests. Prevents large assignments from falling back to a shallow one-shot answer.
Context Pack User notes, brief/rubric notes, uploaded file text, and academic paper context are assembled. Keeps the draft tied to the real assessment and available evidence.
Literature Retrieval Semantic Scholar, CORE, and arXiv feed the external paper pool; OpenAlex is used as fallback. Provides real papers instead of generic filler references.
Drafting The long-form pipeline plans sections, streams status updates, and drafts the paper body. Improves completeness and makes large jobs feel active instead of frozen.
Quality Check Academic answers can go through a mode-specific review pass before final storage. Reduces under-length, weakly structured, or rubric-misaligned output.
Persistence Cited papers are saved as notes when truly used; the final assistant draft is saved to the conversation. Strengthens future retrieval and keeps the conversation auditable.
Streaming and UI

How the Frontend Now Reflects Agent Work

Typed SSE Events

Message deltas, replacement updates, completion metadata, tool status, quizzes, note events, and workflow events now share a typed contract instead of raw ad hoc payloads.

Button-Level Loading

Loading indicators now sit on the action that caused them: send, translate, save-to-note, and quiz-result save. This avoids duplicated chat-level spinners that made the UI noisy.

Workflow Conversations

When a workflow action runs, the user confirmation and the assistant result can be saved back into the conversation so action side effects remain visible in chat history.

Known Next Steps

What Still Needs Further Refactoring

  • chat_service.py is still too large. Routing is cleaner now, but orchestration for long academic drafts, file-upload synthesis, and quality review should be extracted further.
  • Context compression is not implemented yet. Hermes has a mature middle-turn compression model; MemoLink would benefit from a focused version later.
  • Hybrid retrieval can improve again. The current note path is stronger than before, but a clearer BM25 + vector + graph weighting strategy would make retrieval more explainable.
  • More architecture docs can be split out. Teams, email, workflow actions, and evaluation analytics each deserve their own deeper design page if the documentation set continues to grow.
